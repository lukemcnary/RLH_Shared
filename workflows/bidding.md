# Workflow: Bidding

> Status: Authoritative

This workflow turns scoped work into bidder-facing packages and award decisions without making bidding the only path through the product.

---

## Outcome

By the end of bidding:

- project scope is grouped into clear bid packages
- cost items are linked where financial detail already exists
- bidders and quote artifacts are organized
- assignment context is reflected back onto the package and quote records
- the commercial result is tied back to the same structured project records the team planned from

---

## Steps

1. Start from project scope that is clear enough to price and award.
   A bid package should not become the place where ambiguous scope is resolved for the first time.

2. Group the relevant project-scope records into a bid package.
   Package by trade and work scope so the bidder sees one coherent ask. Link cost items too if the financial breakdown already exists.

3. Confirm who should be invited to bid.
   Tie the package to the right trade context and identify the companies being solicited. If no formal bid is needed, the builder may assign the company directly on the package instead.

4. Issue the package externally.
   The shared system holds the structure; supporting quote artifacts can live in SharePoint and linked communication channels as needed. The app should connect that context, not become a second file system of record.

5. Compare returned pricing against package scope and estimate ranges.
   Review whether quotes actually cover the intended work and whether the numbers still make sense relative to the Development-side estimate.

6. Record the award decision back onto the project records.
   Update the bid package and accepted quote so the commercial result is visible in the shared system. Any direct company links on the package or related records should be populated or confirmed here too.

---

## Done When

- each awarded piece of project scope has a visible winning package path
- the PM can trace scoped work from planning to package to accepted quote
- the assignment story is clear without forcing every project through a richer bridge-era trade container
