# UI Design

> Status: Draft

This document defines the visual philosophy, layout approach, and interaction patterns for the combined app's UI surfaces.

---

## Philosophy

The design should feel like a purpose-built construction tool, not a generic enterprise form. The current direction is:

- Calm, not cluttered
- Information hierarchy that mirrors how PMs actually think
- One clear primary action per screen
- Trade-ready output

---

## Layout Patterns

The current layout language centers on:

- List + detail panel
- Full-width workspace
- Focused record view
- Modal / action sheet
